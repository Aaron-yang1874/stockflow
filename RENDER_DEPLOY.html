StockFlow 后端服务 + 数据库 · 一键部署指南

✓ GitHub 已就绪 🔗 Aaron-yang1874/stockflow

📋 部署概览

您需要在 Render 上创建 3 个服务： PostgreSQL 数据库、 NestJS 业务后端、 NestJS 管理后台后端

💰 免费额度说明

• PostgreSQL: 免费 1GB 存储
• Web Service: 免费 750 小时/月（足够小型项目）
• 无需信用卡即可开始使用

🗄️ 步骤 1/3：创建 PostgreSQL 数据库

🚀 一键前往 Render 创建数据库 → 必需
1选择类型
在 Render Dashboard 点击 "New +" → "PostgreSQL"
2配置名称
Name 填写：stockflow-db
3选择计划
Plan 选择：Free（1GB存储，免费！）
4选择区域
Region 选择：Singapore 或离您最近的区域
5数据库名称
Database Name：inventory
6创建并记录
点击 "Create Database"，复制保存连接信息！

⚠️ 重要：请记录以下信息

创建完成后，在数据库详情页找到 Connection 部分，
记录下 Host、Username、Password，
后续配置后端时需要用到！

⚙️ 步骤 2/3：部署业务后端 API

🚀 一键前往 Render 部署业务后端 → 必需

📝 配置详情：

配置项 值 说明
Name stockflow-backend 自动
Runtime Docker 必选
Dockerfile Path ./backend/Dockerfile 修改!
Docker Context ./backend 修改!
Branch main 默认
Plan Free 免费

🔐 环境变量（Environment Variables）：

在 "Advanced" 区域添加以下环境变量，将 <值> 替换为步骤1中记录的实际信息

Key Value 状态
NODE_ENV production 固定
PORT 3000 固定
DB_HOST <数据库 Host> 必填
DB_PORT 5432 固定
DB_USERNAME <数据库 Username> 必填
DB_PASSWORD <数据库 Password> 必填
DB_NAME inventory 固定
REDIS_HOST redis://localhost:6379 可选
JWT_SECRET <生成一个长随机字符串> 必填
CORS_ORIGIN https://stockflow-frontend.vercel.app 前端URL
// JWT_SECRET 快速生成方法（在浏览器控制台运行）： crypto.randomUUID() + crypto.randomUUID()

// 或者访问这个链接在线生成： https://generate-secret.vercel.app/32

🔐 步骤 3/3：部署管理后台后端 API

🚀 一键前往 Render 部署管理后台 → 必需

📝 与业务后端的区别（只有这些不同！）：

配置项 业务后端值 管理后台值（改这里！）
Name stockflow-backend stockflow-admin-backend
Dockerfile Path ./backend/Dockerfile ./admin-backend/Dockerfile
Docker Context ./backend ./admin-backend
PORT 3000 3001
DB_NAME inventory admin_db
CORS_ORIGIN frontend URL https://stockflow-admin-frontend.vercel.app
JWT_SECRET xxx... ⚠ 使用不同的密钥！

⚠️ 重要提醒

管理后台的 JWT_SECRET 必须与业务后端使用不同的值！
其他环境变量（DB_HOST、DB_PASSWORD 等）与业务后端相同。
