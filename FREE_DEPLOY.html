📖 详细教程：Supabase + Glitch 组合（推荐）

🗄️ A. 创建 Supabase 数据库

注册账号
访问 supabase.com，点击 "Start your project"，使用 GitHub 登录（无需信用卡！）
创建新项目
Organization 选默认或新建 → Name 填写 stockflow-db → Database Password 设置一个强密码（记住它！）→ Region 选择 Singapore 或离您最近的
获取连接信息
项目创建后，进入 Settings → Database 页面，记录以下信息：
配置项值的位置示例
DB_HOSTConnection string 中的 host 部分db.xxx.supabase.co
DB_PORT固定值5432
DB_USERNAMEConnection string 中的 userpostgres
DB_PASSWORD您刚才设置的密码your-password-here
DB_NAME固定值postgres
JWT_SECRET自己生成的随机字符串crypto.randomUUID() x2

⚙️ B. 在 Glitch 部署业务后端

导入 GitHub 项目
访问下方按钮，或手动去 glitch.com 点击 "New Project → From GitHub"，搜索 Aaron-yang1874/stockflow
设置项目目录
Root Directory 设为：backend（重要！否则会找错位置）
添加环境变量
在项目 Settings → Environment Variables 中添加上方表格中的所有变量
修改启动命令
Start Command 改为：npm run start:prod
等待构建完成
Glitch 会自动 npm install → nest build → 运行，约 3-5 分钟
👾 立即开始：导入业务后端到 Glitch →

重复步骤 B，但修改以下配置：
